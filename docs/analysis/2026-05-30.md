## PHẦN 1: TÍNH NĂNG TRỢ LÝ DỰ ÁN AI (AI PROJECT ASSISTANT CHATBOT)

### 1. PHẠM VI NGHIÊN CỨU & PHÁT TRIỂN
* **Phạm vi:** **Frontend** (Tạo component trợ lý AI mới, tích hợp tab "Trợ lý AI" vào [ProjectDetail.tsx](file:///c:/project/task-flow/frontend/src/pages/ProjectDetail.tsx)). Không làm thay đổi cơ sở dữ liệu backend, đảm bảo tính bảo mật tuyệt đối cho dữ liệu vì API Key lưu trực tiếp ở phía client và gọi API trực tiếp tới Google AI Studio.
* **Mục tiêu:**
  * Cho phép thành viên/quản trị viên trò chuyện trực tiếp với AI về dự án của mình.

    * *Tóm tắt dự án:* Đọc danh sách tasks, thống kê trạng thái và báo cáo.
    * *Phân tích rủi ro & Blockers:* Tìm các task quá hạn, hoặc các task bị cản trở bởi task dependencies chưa hoàn thành.
    * *Tối ưu phân bổ công việc:* Phân tích số lượng task được giao cho từng member để phát hiện quá tải hoặc thiếu phân công.
  * Đảm bảo giao diện Obsidian dark theme đồng bộ với hệ thống.

### 3. PHƯƠNG ÁN TRIỂN KHAI CHI TIẾT
#### A. Tích hợp Tab mới trong `ProjectDetail.tsx`
1. Thêm import icon `RobotOutlined` từ `@ant-design/icons`.
2. Đăng ký tab mới có key `ai-assistant` trong component `Tabs` tại [ProjectDetail.tsx](file:///c:/project/task-flow/frontend/src/pages/ProjectDetail.tsx):
   ```tsx
   {
     key: 'ai-assistant',
     label: (
       <span className="flex items-center gap-2">
         <RobotOutlined /> Trợ lý AI
       </span>
     ),
     children: (
       <ProjectAiAssistant
         projectId={id}
         project={project}
         tasks={tasks}
         members={members}
       />
     ),
   }
   ```

#### B. Tạo component `ProjectAiAssistant.tsx`
Tạo tệp mới [ProjectAiAssistant.tsx](file:///c:/project/task-flow/frontend/src/components/task/ProjectAiAssistant.tsx):
1. **Kiểm tra khóa API Key:**
   * Lấy `gemini_api_key` từ `localStorage`.
   * Nếu chưa có key, hiển thị một giao diện hướng dẫn người dùng click đi tới trang Cấu hình để thiết lập key của họ trước khi sử dụng.
2. **Cơ cấu Context Prompt (Dữ liệu dự án gửi kèm):**
   Khi người dùng gửi tin nhắn hoặc kích hoạt Quick Action, component sẽ lấy toàn bộ dữ liệu dự án hiện có từ props và cấu trúc một prompt dạng system instructions/context như sau:
   * **Thông tin dự án:** Tên dự án, Mô tả, Trạng thái.
   * **Thành viên:** Danh sách thành viên kèm email và vai trò trong dự án.
   * **Công việc hiện tại:** Danh sách tất cả các task bao gồm tiêu đề, mô tả, trạng thái (TODO, IN_PROGRESS, REVIEW, DONE), độ ưu tiên (LOW, MEDIUM, HIGH, URGENT), người được giao (Assignee), hạn chót (Deadline), ngày bắt đầu (StartDate), và danh sách công việc phụ thuộc (dependencies).
   * **Yêu cầu chỉ thị đối với AI:** 
     * Đóng vai trò là một Trợ lý quản lý dự án (Project Manager Assistant) thông minh, chuyên nghiệp.
     * Cung cấp câu trả lời khách quan, rõ ràng, tập trung vào giải quyết vấn đề và định dạng đẹp bằng Markdown.
     * Trả lời bằng tiếng Việt.
3. **Cơ chế lưu trữ hội thoại:**
   * Hội thoại hiện tại lưu trong React state `messages` để hiển thị trên màn hình.
   * Đồng thời lưu lịch sử trò chuyện của dự án này vào `localStorage` dưới khóa `taskflow_ai_history_${projectId}` để người dùng không bị mất lịch sử khi chuyển tab hoặc tải lại trang.
4. **Gọi API Gemini:**
   * Sử dụng endpoint:
     `https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=${apiKey}`
   * Gửi toàn bộ lịch sử trò chuyện theo đúng cấu trúc API Gemini (`contents: [{ role: "user" | "model", parts: [{ text: string }] }]`) kết hợp với context hệ thống ở tin nhắn đầu tiên hoặc systemInstruction.

## PHẦN 2: NÂNG CẤP TÍNH NĂNG ĐIỀU HƯỚNG VÀ TƯƠNG TÁC THÔNG MINH

### 1. PHẠM VI NGHIÊN CỨU & PHÁT TRIỂN
* **Phạm vi:** **Frontend**
  * Cập nhật bộ kết xuất Markdown [MarkdownRenderer.tsx](file:///c:/project/task-flow/frontend/src/components/common/MarkdownRenderer.tsx) để phân tích các cú pháp liên kết đặc biệt:
    * `task:taskId` -> Mở `TaskDetailModal` xem chi tiết công việc.
    * `tab:tabKey` -> Chuyển sang tab tương ứng trên trang dự án (`overview`, `board`, `calendar`, `timeline`, `members`, `activity`, `charter`, `settings`).
    * `project:projectId` -> Điều hướng người dùng sang trang danh sách dự án (`project:list`) hoặc sang dự án khác (`project:projectId`) qua React Router.
    * `action:actionKey` -> Kích hoạt trực tiếp các chức năng modal/form:
      * `create-task`: Mở modal tạo công việc mới.
      * `export-csv`: Tự động tải xuống báo cáo dữ liệu dự án dạng CSV.
      * `invite-member`: Chuyển sang phân hệ Thành viên.
      * `manage-labels`: Chuyển sang phân hệ Cấu hình dự án (để quản lý nhãn công việc).
  * Cập nhật prompt hệ thống của trợ lý AI [ProjectAiAssistant.tsx](file:///c:/project/task-flow/frontend/src/components/task/ProjectAiAssistant.tsx) để hướng dẫn mô hình sử dụng đúng cú pháp liên kết trên khi đề cập đến công việc hoặc các phân hệ của dự án, đồng thời yêu cầu phản hồi ngắn gọn, súc tích và thể hiện tính điều phối cao.
  * Kết nối các callback từ [ProjectDetail.tsx](file:///c:/project/task-flow/frontend/src/pages/ProjectDetail.tsx) xuống component chatbot.

### 2. THIẾT KẾ PHƯƠNG ÁN TƯƠNG TÁC
Khi AI phản hồi về các công việc quá hạn hoặc bị nghẽn:
* **Trước nâng cấp:**
  * AI: Công việc "API Security Protocol Review" (ID: 3d289521...) đang quá hạn.
* **Sau nâng cấp:**
  * AI: Công việc [API Security Protocol Review](task:3d289521-248d-4340-9821-cd0b6f6aaffa) đang quá hạn. Bạn có thể mở [Bảng công việc Kanban](tab:board) hoặc [Xuất báo cáo CSV](action:export-csv) để kiểm tra, hoặc [Tạo nhiệm vụ mới](action:create-task) để thay thế.

Người dùng click vào liên kết -> Mở modal chi tiết hoặc kích hoạt trực tiếp các chức năng tương ứng mà không cần thao tác thủ công qua nhiều menu.

### 3. PHƯƠNG ÁN TRIỂN KHAI CHI TIẾT
#### A. Cập nhật `MarkdownRenderer.tsx`
1. Bổ sung các props:
   ```typescript
   onTaskClick?: (taskId: string) => void;
   onTabClick?: (tabKey: string) => void;
   onProjectClick?: (projectId: string) => void;
   onActionClick?: (actionKey: string) => void;
   ```
2. Trong hàm `parseInline` tại đoạn parse link `first.type === 'link'`, kiểm tra nếu `m[3]` (URL của link):
   * Bắt đầu bằng `task:`: Gọi `onTaskClick` và thêm biểu tượng `📋` trước text.
   * Bắt đầu bằng `tab:`: Gọi `onTabClick` và thêm biểu tượng `🔗` trước text.
   * Bắt đầu bằng `project:`: Gọi `onProjectClick` và thêm biểu tượng `📁` trước text.
   * Bắt đầu bằng `action:`: Gọi `onActionClick` và kết xuất dưới dạng một **Tag/Nút bấm nổi bật** có biểu tượng `⚡`.

#### B. Cập nhật `ProjectAiAssistant.tsx`
1. Nhận các props tương ứng và truyền chúng xuống `MarkdownRenderer`.
2. Cấu hình lại prompt hệ thống trong `buildProjectContext()` để hướng dẫn mô hình Gemini sinh đúng định dạng Markdown và hướng dẫn ngắn gọn, súc tích bằng tiếng Việt.

#### C. Cập nhật `ProjectDetail.tsx`
Truyền các callback điều hướng thực tế từ page xuống `ProjectAiAssistant`:
* `onTaskClick={(taskId) => { setSelectedTaskId(taskId); setIsDetailOpen(true); }}`
* `onTabClick={(tabKey) => setActiveTab(tabKey)}`
* `onProjectClick={(projId) => navigate(projId === 'list' ? '/projects' : '/projects/' + projId)}`
* `onActionClick={(actionKey) => { ... }}` xử lý mở modal tạo task, xuất báo cáo CSV, chuyển tab tương ứng.
