### 3. PHƯƠNG ÁN TRIỂN KHAI CHI TIẾT
#### A. Database Schema
Cập nhật bảng `Task` trong [schema.prisma](file:///c:/project/task-flow/backend/prisma/schema.prisma) để thêm trường:
```prisma
model Task {
  id          String    @id @default(uuid())
  title       String
  description String?
  status      String    @default("TODO")
  priority    String    @default("MEDIUM")
  startDate   DateTime?
  deadline    DateTime?
  // ... other fields
}
```

#### B. Backend API & Validation
1. Cập nhật `createTaskSchema` và `updateTaskSchema` trong [validation.ts](file:///c:/project/task-flow/backend/src/utils/validation.ts):
   * Cho phép nhận `startDate` dưới dạng ISO String hoặc Date.
   * Ràng buộc: Nếu có cả `startDate` và `deadline`, bắt buộc `startDate` phải nhỏ hơn hoặc bằng `deadline`.
2. Cập nhật `createTask` và `updateTask` trong [taskController.ts](file:///c:/project/task-flow/backend/src/controllers/taskController.ts) để tiếp nhận và lưu trường `startDate`.

#### C. Frontend Types & APIs
1. Cập nhật interface `Task` trong [index.ts](file:///c:/project/task-flow/frontend/src/types/index.ts):
   * Thêm trường `startDate?: string;`.
2. Cập nhật `taskApi.ts` để truyền `startDate` khi gọi API POST/PATCH.

#### D. Frontend UI Components
1. **DatePicker trong Forms:**
   * Cập nhật `TaskFormModal.tsx` thêm trường "Ngày bắt đầu" (DatePicker) cạnh "Hạn chót".
   * Kiểm tra điều kiện: Ngày bắt đầu không được sau Hạn chót.
2. **Hiển thị thông tin:**
   * Cập nhật `TaskDetailModal.tsx` để hiển thị "Ngày bắt đầu" bên dưới / bên cạnh "Hạn chót" ở panel thông tin bên phải.
3. **Biểu đồ Gantt Chart (`TaskTimeline.tsx`):**
   * Tạo component mới [TaskTimeline.tsx](file:///c:/project/task-flow/frontend/src/components/task/TaskTimeline.tsx).
   * Vẽ lưới thời gian cuộn ngang bao gồm header ngày/tuần.
   * Tính toán vị trí trái (`left`) và độ rộng (`width`) của thanh công việc dựa trên tỷ lệ ngày bắt đầu, hạn chót so với tầm nhìn thời gian của biểu đồ (ví dụ: mặc định 30 ngày quanh khoảng thời gian các công việc diễn ra).
   * Vẽ liên kết dependencies bằng SVG `<svg className="absolute inset-0 pointer-events-none">` vẽ thẻ `<path d="..." />` nối giữa các thanh công việc.
   * Tích hợp bộ lọc (Tên công việc, Người thực hiện, Trạng thái) và cho phép bấm trực tiếp vào thanh công việc / hàng công việc để mở `TaskDetailModal`.
4. **Tích hợp tab mới:**
   * Cập nhật `ProjectDetail.tsx` để thêm tab "Timeline" bên cạnh các tab Overview, Board, Members, Settings.
   * Đưa component `TaskTimeline` vào tab mới này.

### 3. PHƯƠNG ÁN TRIỂN KHAI CHI TIẾT
#### A. Frontend Routes & Sidebar Integration
1. Đăng ký Route mới trong [App.tsx](file:///c:/project/task-flow/frontend/src/App.tsx):
   ```tsx
   <Route path="/help" element={<Help />} />
   ```
2. Thêm MenuItem mới vào sidebar trong [DashboardLayout.tsx](file:///c:/project/task-flow/frontend/src/layouts/DashboardLayout.tsx):
   * Sử dụng icon `QuestionCircleOutlined` từ `@ant-design/icons`.
   * Thêm item vào `menuItems` và cập nhật cơ chế sinh breadcrumbs tự động.

#### B. Help Page Component (`Help.tsx`)
Tạo component mới [Help.tsx](file:///c:/project/task-flow/frontend/src/pages/Help.tsx):
1. **Quản lý State:**
   * `activeKey`: Danh mục tài liệu đang được chọn.
   * `searchText`: Từ khóa tìm kiếm tài liệu.
2. **Bộ lọc tìm kiếm tài liệu:**
   * Khi người dùng gõ từ khóa vào thanh tìm kiếm, hệ thống sẽ lọc các bài viết hoặc câu hỏi FAQ có chứa từ khóa đó và tự động làm nổi bật (highlight).
3. **Bảng phân quyền (RBAC Privilege Matrix):**
   * Sử dụng component `Table` của Antd hoặc render HTML table với viền Slate mỏng.
   * Cột: Tính năng, Quản trị viên (Admin), Trưởng nhóm (Leader), Thành viên (Member).
   * Dùng icon `CheckOutlined` (màu xanh lá) và `CloseOutlined` (màu đỏ) đại diện cho quyền.
4. **FAQ Accordion:**
   * Sử dụng component `Collapse` của Ant Design để hiển thị các câu hỏi thường gặp:
     * *Làm sao để thay đổi vai trò của thành viên khác?* (Chỉ Leader hoặc Admin thực hiện trong tab Cấu hình dự án).
     * *Tại sao tôi không kéo thả được công việc sang Done?* (Kiểm tra nếu công việc đó có các công việc phụ thuộc chưa hoàn thành).
     * *Tôi có thể tạo dự án cá nhân không?* (Thành viên thông thường có thể tự tạo dự án cá nhân qua Tab "Dự án của tôi").
