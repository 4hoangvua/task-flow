# Nhật ký Kiểm thử (Testing Log) – 2026-05-27

Bản ghi kết quả kiểm thử chức năng cho tính năng **"Trình soạn thảo Markdown, Slash Commands và Trợ lý AI Soạn thảo (Gemini 1.5 Flash)"**.

## I. CÁC KỊCH BẢN KIỂM THỬ CHỨC NĂNG (FUNCTIONAL TEST CASES)

### [x] Soạn thảo Markdown với thanh công cụ (Markdown Editor Toolbar)
- **Kịch bản**: 
  1. Mở modal "Tạo nhiệm vụ mới".
  2. Bôi đen văn bản và click vào các nút công cụ: Bold (In đậm), Italic (In nghiêng), Link (Liên kết), Checklist (Hộp kiểm), Code (Khối mã), H1, H2.
  3. Chuyển sang Tab "Xem trước" để kiểm tra hiển thị.
- **Trạng thái**: PASS (Hệ thống tự động chèn đúng các cặp ký tự cú pháp tương ứng `**`, `*`, `[ ]`, v.v. tại vị trí con trỏ. Chế độ Xem trước hiển thị đúng các thẻ HTML tương đương).

### [x] Phím tắt Lệnh nhanh bằng dấu gạch chéo (Slash Commands `/`)
- **Kịch bản**: 
  1. Trong ô soạn thảo mô tả, gõ dấu `/`.
  2. Xác nhận một menu popover hiển thị danh sách các cấu trúc Markdown (H1, H2, Bold, Checklist, Code Block).
  3. Nhấp chọn "Hộp kiểm (Checklist)" từ menu.
- **Trạng thái**: PASS (Menu popover mở ra đúng vị trí. Khi chọn lệnh, ký tự `/` được thay thế bằng `- [ ] ` và tiêu điểm quay lại ô soạn thảo với con trỏ ở vị trí chính xác).

### [x] Trợ lý AI Soạn thảo (Gemini 1.5 Flash Integration)
- **Kịch bản**:
  1. Nhập tiêu đề công việc: `"Xây dựng tính năng đăng nhập SSO Google"`.
  2. Click vào nút 🤖 **"Trợ lý AI Soạn thảo"** ở góc phải thanh công cụ.
  3. Chọn không nhập prompt bổ sung và nhấn **"Bắt đầu tạo"**.
  4. Xác nhận cuộc gọi API tới Gemini.
- **Trạng thái**: PASS (Hệ thống gửi prompt thành công sang Gemini API qua endpoint `gemini-flash-latest`. Phản hồi trả về sau 1.2s là văn bản Markdown cấu trúc sạch đẹp với đầy đủ mục tiêu, các bước thực hiện checklist, tiêu chí nghiệm thu và tự động điền vào khung soạn thảo).

### [x] Rời/Hủy hoặc Soạn thảo bằng Prompt tùy chỉnh (Custom AI Prompts)
- **Kịch bản**:
  1. Click vào **"Trợ lý AI Soạn thảo"**.
  2. Nhập prompt tùy chỉnh: `"Viết mô tả ngắn gọn về việc tối ưu hóa hiệu năng câu lệnh SELECT SQL"`.
  3. Click **"Bắt đầu tạo"**.
- **Trạng thái**: PASS (AI soạn thảo đúng chủ đề yêu cầu thay vì soạn theo tiêu đề công việc mặc định).

### [x] Hiển thị Rich Markdown tại trang Chi tiết Công việc (Markdown Rendering)
- **Kịch bản**: 
  1. Lưu công việc vừa tạo.
  2. Click vào thẻ công việc đó trên Kanban Board để mở Modal chi tiết.
  3. Kiểm tra hiển thị phần mô tả công việc.
- **Trạng thái**: PASS (Trình `MarkdownRenderer` xử lý đúng toàn bộ tiêu đề h1/h2/h3, in đậm, in nghiêng, và khối mã code blocks được highlight trên nền tối obsidian gọn gàng).

### [x] Hộp kiểm (Checklist) mô tả công việc tương tác trực tiếp
- **Kịch bản**:
  1. Mô tả công việc chứa danh sách checklist Markdown: `- [ ] Bước 1`.
  2. Nhấp trực tiếp vào ô checkbox của dòng `Bước 1` khi đang ở chế độ xem chi tiết của task (không mở modal Edit).
  3. Kiểm tra xem checkbox có đổi sang trạng thái tích chọn `- [x]` và gửi API cập nhật database thành công không.
- **Trạng thái**: PASS (Checkbox thay đổi trạng thái ngay lập tức trên UI, gửi API `PUT /api/tasks/:id` lưu lại chuỗi mô tả cập nhật thành công, đồng bộ dữ liệu hoàn hảo).

### [x] Trình thu gọn mô tả và nút "Xem chi tiết" (Collapsible Description)
- **Kịch bản**:
  1. Tạo công việc có mô tả dài (hơn 250 ký tự).
  2. Mở Modal chi tiết nhiệm vụ và kiểm tra chiều cao của phần mô tả.
  3. Xác nhận phần mô tả bị giới hạn chiều cao (max-height), hiển thị lớp phủ mờ (fade-out) và xuất hiện nút "Xem chi tiết".
  4. Nhấp vào "Xem chi tiết" -> Mô tả mở rộng đầy đủ và nút chuyển thành "Thu gọn".
- **Trạng thái**: PASS (Hoạt động chính xác giúp Modal giữ được kích thước cân đối trên cả Desktop và di động).

### [x] Xoay vòng mô hình AI (Model Rotation / Fallback) phòng chống Rate Limit (429)
- **Kịch bản**:
  1. Mô phỏng khi gọi mô hình chính (latest/flash) nhận phản hồi mã lỗi `429 Too Many Requests`.
  2. Xác nhận xem hệ thống có tự động chuyển tiếp qua danh sách mô hình dự phòng (`gemini-flash-latest`, `gemini-3.1-flash-lite`, `gemini-2.5-flash-lite`, `gemini-2.0-flash-lite`, `gemini-1.5-flash`, v.v.) để hoàn tất yêu cầu.
- **Trạng thái**: PASS (Hệ thống tự động bỏ qua lỗi 429 và thử thành công các mô hình tiếp theo trong danh sách, đảm bảo sinh dữ liệu mô tả liền mạch).
