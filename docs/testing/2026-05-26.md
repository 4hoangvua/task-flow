### [x] Ràng buộc Tuần hoàn & Tự tham chiếu (Circular & Self-Dependency Validation)
- **Kịch bản**:
  1. Thử thiết lập một task phụ thuộc vào chính nó.
  2. Thiết lập Task A phụ thuộc vào Task B, sau đó thử thiết lập Task B phụ thuộc ngược lại Task A.

### [x] Ngăn chặn Phụ thuộc tuần hoàn Gián tiếp (Indirect Circular Dependency Prevention)
- **Kịch bản**:
  1. Thiết lập Task A phụ thuộc vào Task B, và Task B phụ thuộc vào Task C (A -> B -> C).
  2. Mở chi tiết Task C, kiểm tra ô chọn "Công việc tiên quyết".
  3. Xác nhận Task A không xuất hiện trong danh sách lựa chọn của Task C để ngăn chặn vòng lặp gián tiếp.
  4. Gửi yêu cầu API trực tiếp POST `/api/tasks/:id/dependencies` cố tình thiết lập C phụ thuộc vào A (C -> A).
- **Trạng thái**: PASS (Frontend lọc bỏ Task A khỏi dropdown của Task C; Backend chặn trực tiếp và trả về lỗi 400 cùng thông báo phụ thuộc tuần hoàn chi tiết).

### [x] Ngăn chặn Chuyển trạng thái sang DONE khi Prerequisite chưa hoàn thành
- **Kịch bản**:
  1. Thiết lập Task B phụ thuộc vào Task A. Task A đang ở trạng thái TODO.
  2. Thử chuyển trạng thái Task B sang DONE (bằng cách cập nhật trạng thái trong Modal chi tiết hoặc kéo thả trên Kanban Board).

### [x] Cho phép hoàn thành khi tất cả Prerequisites đạt trạng thái DONE
