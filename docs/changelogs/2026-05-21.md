# Changelog – 2026-05-21

Ngày đầu tiên hoàn thành hệ thống giao diện và cấu hình styling. Dưới đây là toàn bộ các bản sửa, cấu hình, và tối ưu UI đã thực hiện.

---

## Cấu hình Tailwind CSS v4 & Ant Design

- **Vấn đề:** Tailwind CSS v4 không dùng `tailwind.config.js`. Frontend hiển thị sai do Tailwind classes không được compile.
- **Sửa:**
  - Cài đặt `@tailwindcss/vite` plugin.
  - Đăng ký plugin trong `vite.config.ts`.
  - Khai báo `@import "tailwindcss";` tại `index.css`.
- **Quy tắc ghi nhớ:** Tailwind v4 = CSS-first config. KHÔNG tạo `tailwind.config.js`.

## CORS – Dynamic Port Matching

- **Vấn đề:** Vite chạy trên port 5174 (do 5173 bận), gây lỗi CORS `Access-Control-Allow-Origin` mismatch.
- **Sửa:** Cập nhật CORS backend sử dụng regex `/^http:\/\/localhost:\d+$/` để cho phép tất cả các port localhost động.
- **Quy tắc ghi nhớ:** KHÔNG hardcode `CLIENT_URL` làm origin duy nhất.

## Container & Responsive Layout

- **Vấn đề:** `#root` có `max-width: 1126px` cố định và `text-align: center`, gây giao diện bị thu hẹp trên màn hình lớn và văn bản bị canh giữa.
- **Sửa:**
  - Xóa `max-width`, `border-inline`, `text-align: center` trong `index.css` trên `#root`.
  - Cho `#root` chạy full-width (100%).
  - Kiểm tra responsive trên mobile/tablet/desktop.
- **Quy tắc ghi nhớ:** Không đặt kích thước cố định trên `#root`. Để layout components tự quản lý width.

## Port Conflict (EADDRINUSE)

- **Vấn đề:** Backend crash do port 5000 đã bị chiếm bởi tiến trình cũ.
- **Sửa:** Kill tiến trình cũ trước khi chạy `npm run dev`.
- **Quy tắc ghi nhớ:** Kiểm tra background tasks trước khi start server. KHÔNG chạy `npm run build` ở frontend.

## Phân quyền hiển thị giao diện (RBAC UI)

- **Vấn đề:** Giao diện hiển thị các nút/tab mà người dùng MEMBER không có quyền sử dụng, gây lỗi API 403.
- **Sửa:**
  - `Projects.tsx`: Ẩn nút "Tạo dự án mới" cho MEMBER.
  - `ProjectDetail.tsx`: Ẩn tab "Cấu hình" cho non-leader. Ẩn form mời thành viên. Ẩn cột "Hành động" trong bảng member. Thay Select vai trò bằng Tag tĩnh cho non-leader.
  - `TaskBoard.tsx`: Ẩn nút "Thêm công việc". Chặn drag-drop thay đổi status cho non-assignee/non-leader (hiển thị error toast).
  - `TaskDetailModal.tsx`: Ẩn nút "Chỉnh sửa" và "Xóa" cho non-leader. Disable dropdown status cho non-assignee/non-leader. Cho leader xóa bình luận của bất kỳ ai.
  - `useTasks.ts`: Thêm `message.error` khi reorder API thất bại.
- **Quy tắc ghi nhớ:** Tính `isProjectLeader` = `currentMember?.role === 'LEADER' || currentUser?.role === 'ADMIN' || project.ownerId === currentUser?.id`. Truyền prop này xuống các component con.

## Seed Data – Owner phải là Member

- **Vấn đề:** Chủ dự án không hiển thị trong danh sách thành viên vì thiếu record `ProjectMember`.
- **Sửa:** `seed.ts` thêm owner vào project với vai trò LEADER.
